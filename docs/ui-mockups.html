Wie die Ein-Taster-Bedienung auf dem Display aussieht

Jeder Bildschirmzustand aus ui.cpp: die Kanal-Abfrage beim Boot, die Ruhe-Anzeige mit ihrer nach Signalstärke sortierten Fahrerliste (allein / 1 / 2 / 3 Spalten, stumm, Akku-Hinweis), das Sende-Menü, eine eingehende Warnung, der „Noch abgerissen"-Prompt, die Einstellungen (Stumm/Statistik/Ton/Anzeige/Empfindlich/Kanal/Name/Test — Ton und Empfindlichkeit mit der Lineal-Skala 0–10) und der Reichweiten-Test — so wie Ui::render() sie tatsächlich zeichnet. Jede Zeile sitzt hier an der echten drawStr()-Baseline aus dem Code, umgerechnet auf Prozent der 64px-Höhe; nur die Schrift ist zur besseren Lesbarkeit im Chat ein System-Monospace statt der pixelgenauen U8g2-Bitmapfont. Das verbaute 0,96"-Panel ist ein Zweifarb-OLED: Die oberen 16 Pixelzeilen leuchten fest gelb, der Rest blau — Kopfzeile und Trennlinie liegen deshalb bewusst im gelben Streifen, alle Listen und Inhalte beginnen erst ab Pixelzeile 16.
